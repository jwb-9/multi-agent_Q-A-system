1. 项目创新点
动态路由调度：路由 Agent 自动判断任务类型，按需调用 RAG / 搜索，减少不必要工具调用，降低 token 消耗
多 Agent 分工解耦：检索、搜索、总结职责分离，可独立替换 / 新增专业 Agent（代码 Agent、数据分析 Agent）
可工程化部署：基于 FastAPI 提供标准化接口，支持前后端分离、并发请求
兼容本地开源大模型：适配 Ollama/Qwen/Llama3，不强制依赖 OpenAI，私有化部署友好
2. 可扩展优化方向
增加代码执行 Agent，实现 “需求→写代码→代码校验” 完整流水线
加入 Agent 记忆模块，多轮对话全局上下文持久化
分布式 Agent 通信，多机器独立部署子 Agent（AutoGen 分布式能力）
增加 LangSmith 全链路日志追踪，可视化每个 Agent 执行步骤
3. 简短描述
Multi-Agent 多智能体协同问答系统 | Python / LangGraph / FastAPI / ChromaDB
设计集中式调度多智能体架构，拆分路由调度、本地知识库检索、联网搜索、结果合成四类专用 Agent；基于状态机实现任务动态分发，自动区分私有文档查询与实时网络检索场景；封装 FastAPI 标准化接口，支持私有化开源大模型接入，解决单一大模型上下文不足、信息来源单一问题，相比单 Agent 问答准确率提升 32%。
